# AURA v1.0 Smoke Test Checklist

Run these checks before any production deployment or investor demo.

---

## 1. Auth Tests

- [ ] Logged-out user sees only `AuthLanding` — no sidebar, no app data
- [ ] Sign in with email + password works
- [ ] Sign in with magic link works (if enabled)
- [ ] Sign out clears session and returns to AuthLanding
- [ ] Refreshing page while logged in keeps session active
- [ ] API routes return 401 for unauthenticated requests

---

## 2. Wardrobe Tests

- [ ] Add wardrobe item with all fields — item appears in grid
- [ ] Image upload (JPEG/PNG) shows preview and uploads to Supabase Storage
- [ ] HEIC image converts to JPEG on upload (Apple devices)
- [ ] Vision AI analysis runs and pre-fills fields (or shows fallback)
- [ ] AI metadata saved with item and visible on item card
- [ ] Wardrobe persists across page reload
- [ ] Empty state shown when wardrobe is empty
- [ ] Valid items appear in Daily Briefing outfit recommendation

---

## 3. Vision AI Tests

- [ ] Upload wardrobe image → Vision AI analyzes → returns category, color, style
- [ ] AI-suggested fields are highlighted/marked
- [ ] If OpenAI key missing → fallback to mock (no crash)
- [ ] Unsupported image formats show clear error, not a crash
- [ ] ai_metadata is stored in Supabase wardrobe_items row

---

## 4. Daily Briefing Tests

- [ ] Home loads with time-appropriate greeting
- [ ] Weather shown for user's city (or graceful "unavailable" state)
- [ ] Daily outfit recommendation generated from wardrobe
- [ ] Outfit score and breakdown shown (occasion fit, weather fit, style, color)
- [ ] Accept → outfit saved, wears incremented, feedback dispatched
- [ ] Reject → new outfit generated, feedback dispatched
- [ ] Edit → inline swap from alternatives
- [ ] If wardrobe < 2 valid items → empty state shown with CTA
- [ ] Onboarding checklist appears and reflects real state
- [ ] All 6 checklist items check correctly when their conditions are met
- [ ] "AURA is ready for daily use" shown when all 6 complete

---

## 5. Style DNA Tests

- [ ] Settings → Recompute Style DNA runs without error
- [ ] Style DNA profile saved to Supabase `style_dna_profiles`
- [ ] Analytics view shows Style DNA panel when confidence > 0
- [ ] Analytics shows "Building" state when confidence = 0
- [ ] Style DNA influences Daily Briefing outfit scoring

---

## 6. Packing Tests

- [ ] Create trip plan form submits with all fields
- [ ] City lookup auto-detects country and coordinates
- [ ] Weather fetched for destination (or graceful unavailable)
- [ ] Packing list generated with priority tiers
- [ ] Daily outfit plan shown for trip days
- [ ] Missing items panel appears when wardrobe gaps detected
- [ ] Trip plan saved to Supabase `trip_plans`
- [ ] Existing trip plans load on page refresh
- [ ] Delete trip plan works

---

## 7. Occasion Tests

- [ ] Create occasion event with all fields
- [ ] City lookup auto-detects country
- [ ] Get Outfit Recommendation → calls `/api/occasions/recommend`
- [ ] Recommendation shows scores, items, reasoning, risks
- [ ] Accept outfit → status changes to "accepted", feedback dispatched
- [ ] Reject outfit → status changes to "rejected"
- [ ] Events saved to Supabase `occasion_events`
- [ ] Events load on page refresh
- [ ] Weekly brief shows correct count of prepared vs unprepared
- [ ] HomeView shows today/tomorrow events in occasion strip
- [ ] Delete event works

---

## 8. Inspiration Tests

- [ ] Enter item name + category + color + price → AI analysis runs
- [ ] BUY/WAIT/SKIP decision shown with reasoning
- [ ] Score breakdown shown (compatibility, style match, wardrobe impact)
- [ ] Inspiration saved to Supabase `inspiration_items`
- [ ] Add to wardrobe from inspiration works
- [ ] If OpenAI key missing → fallback to mock (no crash)

---

## 9. Analytics Tests

- [ ] Empty state shown when wardrobe is empty
- [ ] KPI grid shows correct values from wardrobe
- [ ] Category coverage table is accurate
- [ ] Wardrobe gaps correctly identified
- [ ] Duplicate risk detection works
- [ ] Outfit feedback stats reflect real feedback events
- [ ] Style DNA panel shown when confidence > 0
- [ ] Style DNA "Building" state shown when confidence = 0

---

## 10. Settings / Release Tests

- [ ] Settings form saves and persists (name, city, style goal, budget)
- [ ] Style DNA recompute button works from Settings
- [ ] Release card shows `v1.0.0 Preview`
- [ ] Release notes list the v1.0 features correctly
- [ ] Location detection (browser geolocation) works or gracefully falls back

---

## 11. Mobile Tests

Test at 390px width (iPhone Pro) and 768px width (iPad):

- [ ] No horizontal overflow on any view
- [ ] Navigation is usable (all 8 items visible / scrollable)
- [ ] Cards stack cleanly in single column
- [ ] Buttons are tappable (min 44px touch target)
- [ ] Forms are readable and inputs are not too small
- [ ] Recommendation panels fit screen without clipping
- [ ] Table rows don't cause overflow (horizontal scroll on table)
- [ ] AuthLanding shows only form panel (hero hidden on mobile)
- [ ] Toast notifications are visible (bottom right, above nav)

---

## 12. Production Environment Checklist

Before deploying to production:

- [ ] `SUPABASE_URL` set server-side
- [ ] `SUPABASE_ANON_KEY` set (public is fine — RLS enforces ownership)
- [ ] `OPENAI_API_KEY` set server-side only (NOT as NEXT_PUBLIC_*)
- [ ] `WEATHER_API_KEY` set server-side only (NOT as NEXT_PUBLIC_*)
- [ ] `NEXT_PUBLIC_ENABLE_DEMO_TOOLS` is NOT set or is `false` in production
- [ ] `AI_PROVIDER` set to `openai` (or desired provider) in production
- [ ] RLS policies active on all tables:
  - [ ] `wardrobe_items`
  - [ ] `inspiration_items`
  - [ ] `saved_outfits`
  - [ ] `style_dna_profiles`
  - [ ] `trip_plans`
  - [ ] `occasion_events`
  - [ ] `feedback_events`
  - [ ] `orders`
  - [ ] `stylist_bookings`
- [ ] v0.9 migration SQL run: `supabase/v0.9_migration.sql`
- [ ] Build passes: `npm run build`
- [ ] No TypeScript errors: `npm run typecheck`
- [ ] No lint errors: `npm run lint`

---

## 13. Demo Mode Test

With `NEXT_PUBLIC_ENABLE_DEMO_TOOLS=true`:

- [ ] "Load Demo Data" button visible in topbar
- [ ] Clicking loads: wardrobe (11 items), 2 inspirations, 2 occasions, 1 trip plan, 1 saved outfit
- [ ] All data is realistic — no "test", "aaa", placeholder garbage
- [ ] App looks investor-demo ready after loading
- [ ] "Clear Data" button resets all state
- [ ] Demo tools NOT visible when flag is absent or false
